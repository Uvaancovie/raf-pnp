# Contributing

Thank you for your interest in contributing to RAF PNP. To keep contributions fast and consistent, please follow these guidelines:

- Fork the repository and create a feature branch from `main`.
- Use clear, small commits and descriptive commit messages.
- Open a pull request (PR) against `main` with a clear summary of changes and testing instructions.
- Add or update tests for bug fixes and new features.
- Ensure the CI pipeline passes before requesting review.
- Follow the coding style in the repository and run `dotnet format` before committing.

For larger changes, open an issue to discuss the design before implementing.
